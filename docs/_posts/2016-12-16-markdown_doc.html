概述

撰写网页内容是个很无聊的，这是个公认的事实，WYSIWYG编辑器对这项工作很有帮助， 但是它生成的代码往往很紊乱，有时候甚至造成页面丑的要你命。 用markdown来书写则是一个更好的选择。它的哲学是易读易写，没有什么复杂的东西，也不会生成丑陋的代码。

区块元素

标题

markdown有两种定义标题的方式: 类Setext和类atx形式

  1. 类Setext

    类Setext形式是用底线的形式，利用=(最高级标题)和-(第二级标题): 

    这是高级标题
==========

这是二级标题
----------
    效果如下: 
    <h1>这是高级标题</h1>
<h2>这是二级标题</h2>

  2. 类atx

    类`atx`形式是在行首插入1到6个#，对应到标题1到6级: 

    # 一级标题
## 二级标题
### 三级标题
#### 四级标题
##### 五级标题
###### 六级标题
    效果如下: 
    <h1>一级标题</h1>
<h2>二级标题</h2>
<h3>三级标题</h3>
<h4>四级标题</h4>
<h5>五级标题</h5>
<h6>六级标题</h6>

段落和换行

段落由一个或多个连续的文本行组成，前后要有一个以上的空行: 


这是一个段落。

这里是另外一个段落。
效果如下: 
<p>这是一个段落。</p>
<p>这里是另外一个段落。</p>

如果要依赖 Markdown 来插入 <br> 标签的话，在插入处先按入两个以上的空格然后回车: 

这里需要换行  ↵
效果如下: 
这里需要换行<br>

引用区块

在整个段落的第一行最前面加上 >: 

>This is a blockquote with two paragraphs.
Lorem ipsum dolor sit amet,consectetuer adipiscing elit.
Aliquam hendrerit mi posuere lectus.
Vestibulum enim wisi, viverra nec, fringilla in,
laoreet vitae, risus.
效果如下: 
<blockquote>
  <p>
    This is a blockquote with two paragraphs.
    Lorem ipsum dolor sit amet,consectetuer adipiscing elit.
    Aliquam hendrerit mi posuere lectus.
    Vestibulum enim wisi, viverra nec, fringilla in,
    laoreet vitae, risus.
  </p>
</blockquote>
引用区块支持嵌套: 
> Donec massa lacus, ultricies a ullamcorper in, fermentum sed augue.
Nunc augue augue, aliquam non hendrerit ac, commodo vel nisi.
>> Sed adipiscing elit vitae augue consectetur a gravida nunc vehicula.
Donec auctorodio non est accumsan facilisis.
Aliquam id turpis in dolor tincidunt mollis ac eu diam.
效果如下: 
<blockquote>
  <p>Donec massa lacus, ultricies a ullamcorper in, fermentum sed augue.
  Nunc augue augue, aliquam non hendrerit ac, commodo vel nisi.</p>
  <blockquote>
    <p>Sed adipiscing elit vitae augue consectetur a gravida nunc vehicula.
    Donec auctorodio non est accumsan facilisis.
    Aliquam id turpis in dolor tincidunt mollis ac eu diam.</p>
  </blockquote>
</blockquote>
引用的区块内也可以使用其他的Markdown语法，包括标题、列表、代码区块等。

代码区块

和程序相关的写作或是标签语言原始码通常会有已经排版好的代码区块， 通常这些区块我们并不希望它以一般段落文件的方式去排版，而是照原来的样子显示， Markdown 会用 <pre> 和 <code> 标签来把代码区块包起来。

  1. 缩进实现

    要在 Markdown 中建立代码区块只要简单地缩进 4 个空格或是 1 个制表符就可以: 

        // Some comments
    line 1 of code
    line 2 of code
    line 3 of code
    效果如下: 
    <pre>
  <code>
    // Some comments
    line 1 of code
    line 2 of code
    line 3 of code
  </code>
</pre>

  2. 栅栏实现

    可以使用```实现多行代码引用: 

    ``` markup
Sample text here...
```
    效果如下: 
    <pre>
  <code>Sample text here...</code>
</pre>

列表

  1. 无序列表

    可以使用这三种方法创建无序列表: 

    * valid bullet
- valid bullet
+ valid bullet
    例如: 
    + Lorem ipsum dolor sit amet
+ Consectetur adipiscing elit
+ Integer molestie lorem at massa
+ Facilisis in pretium nisl aliquet
+ Nulla volutpat aliquam velit
  - Phasellus iaculis neque
  - Purus sodales ultricies
  - Vestibulum laoreet porttitor sem
  - Ac tristique libero volutpat at
+ Faucibus porta lacus fringilla vel
+ Aenean sit amet erat nunc
+ Eget porttitor lorem
    效果如下: 
    <ul>
  <li>Lorem ipsum dolor sit amet</li>
  <li>Consectetur adipiscing elit</li>
  <li>Integer molestie lorem at massa</li>
  <li>Facilisis in pretium nisl aliquet</li>
  <li>Nulla volutpat aliquam velit
    <ul>
      <li>Phasellus iaculis neque</li>
      <li>Purus sodales ultricies</li>
      <li>Vestibulum laoreet porttitor sem</li>
      <li>Ac tristique libero volutpat at</li>
    </ul>
  </li>
  <li>Faucibus porta lacus fringilla vel</li>
  <li>Aenean sit amet erat nunc</li>
  <li>Eget porttitor lorem</li>
</ul>

  2. 有序列表

    1. Lorem ipsum dolor sit amet
2. Consectetur adipiscing elit
3. Integer molestie lorem at massa
4. Facilisis in pretium nisl aliquet
    效果如下: 
    <ol>
  <li>Lorem ipsum dolor sit amet</li>
  <li>Consectetur adipiscing elit</li>
  <li>Integer molestie lorem at massa</li>
  <li>Facilisis in pretium nisl aliquet</li>
</ol>

分割线

创建<hr>分割线的方法有这几种: 

___
---
***

行内元素

强调

  1. strong

    **rendered as bold text**
    效果如下: 
    <strong>rendered as bold text</strong>

  2. em

    _rendered as italicized text_
    效果如下: 
    <em>rendered as italicized text</em>

  3. del

    ~~Strike through this text.~~
    效果如下: 
    <del>Strike through this text.</del>

行内代码

`<section></section>`
效果如下: 
<code><section></section></code>

链接

  • 基本形式

    [Assemble](http://assemble.io)
    效果如下: 
    <a href="http://assemble.io">Assemble</a>

  • 添加title属性

    [Upstage](https://github.com/upstage/ "Visit Upstage!")
    效果如下: 
    <a href="https://github.com/upstage/" title="Visit Upstage!">Upstage</a>

  • 命名锚点

    # Table of Contents
  * [Chapter 1](#chapter-1)
  * [Chapter 2](#chapter-2)
  * [Chapter 3](#chapter-3)
    这下链接将跳至一下的锚点上: 
    ## Chapter 1 <a id="chapter-1"></a>
Content for chapter one.

## Chapter 2 <a id="chapter-2"></a>
Content for chapter one.

## Chapter 3 <a id="chapter-3"></a>
Content for chapter one.

图片

Markdown 使用一种和链接很相似的语法来标记图片，允许两种样式： 行内式和参考式

  1. 行内式

    ![Alt text](http://octodex.github.com/images/stormtroopocat.jpg "The Stormtroopocat")
    效果如下: 
    <img src="http://octodex.github.com/images/stormtroopocat.jpg" alt="Alt text" title="The Stormtroopocat">

  2. 参考式

    ![Alt text][id]
[id]: http://octodex.github.com/images/dojocat.jpg  "The Dojocat"
    效果如下: 
    <img src="http://octodex.github.com/images/dojocat.jpg" alt="Alt text" title="The Dojocat">

转义

Markdown 支持以下这些符号前面加上反斜杠来帮助插入普通的符号: 

\   反斜线
`   反引号
*   星号
_   底线
{}  花括号
[]  方括号
()  括弧
#   井字号
+   加号
-   减号
.   英文句点
!   惊叹号

代码高亮

GFM(GitHub Flavored Markdown)可以支持语法高亮。 使用```language就可以自动实现代码高亮的渲染。但这需要插件来支持。
